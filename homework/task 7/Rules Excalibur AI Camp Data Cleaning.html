Welcome to the Data Cleaning Challenge!

In this challenge, you’ll step into the shoes of a data engineer hired by an environmental research institute on marketing.

Your goal: Consolidate, clean, and standardize these datasets to create a unified, analysis-ready dataset. You’ll tackle issues such as mixed formats, duplicate entries, inconsistent units, and varying data frequencies.

Overview

This dataset presents common real-world challenges in data cleaning. It includes:

  • Multiple files in varying formats (.csv, .tsv, .json, etc.)
  • Inconsistent data types and column names across files
  • Different data frequencies (e.g., hourly, monthly)
  • Duplicates and missing entries
  • Much Much More
Data
Dataset Description

This dataset contains customer, employee, product, and transaction data for a retail company, as well as sensor data from the company's stores and error logs related to data processing. This challenge aims to work with the various data files, clean and preprocess the data, and build models to gain insights into the company's operations.

Files

The dataset includes the following files:

customers.csv

  • Description: Information about the company's customers, including personal details, contact information, and account history.
  • Columns: customer_id, name, birth_date, age, email, phone, address (street, city, state, country, postal_code), registration_date, last_login, status, language_preference.

employees.csv

  • Description: Information about the company's employees, such as personal details, job roles, and performance metrics.
  • Columns: employee_id, personal_info (name, birth_date, age, salary, email, phone, address), position, department, performance_metrics.

error_logs.csv

  • Description: Records of various errors and issues that occurred during data processing and system operations.
  • Columns: timestamp, error_type, message.

products.json

  • Description: The company's product catalog, including details about each product.
  • Fields: product_id, name, category, description, specifications (weight, dimensions, temperature_requirements), price, stock_level, supplier_id.

sensor_stores_transactions.zip

  • Description: A compressed folder containing three files related to the company's stores and transactions.
    1. sensor_data.csv
      • Description: Sensor measurements from the company's stores, including temperature, humidity, foot traffic, and energy consumption.
      • Columns: timestamp, store_id, sensor_id, temperature, humidity, foot_traffic, energy_consumption.
    1. stores.csv
      • Description: Information about the company's store locations, including address, size, and performance metrics.
      • Columns: store_id, name, location (coordinates, address), size, opening_hours, manager_id, performance_metrics (revenue, foot_traffic, conversion_rate).
    1. transactions.csv
      • Description: Details about the company's customer transactions, including the items purchased.
      • Columns: transaction_id, customer_id, store_id, timestamp, items (product_id, quantity, unit_price, discount), total_amount, payment_method, loyalty_points.

Data Characteristics

  • The data contains a mix of structured (CSV, JSON) and semi-structured (CSV in a ZIP file) formats.
  • There are various data quality and consistency issues, such as:
    • Inconsistent date and time formats
    • Missing values represented in different ways (NULL, NA, -1, etc.)
    • Ambiguous or contradictory information (e.g., age vs. birth date)
    • Variations in units of measurement (metric vs. imperial)
    • Inconsistent naming conventions and abbreviations
    • Potential outliers and erroneous data points
  • The dataset spans multiple business domains, including customer data, product catalog, store operations, and transaction history.
  • There are relationships between the different data files that participants should identify and leverage for analysis.
  • The error logs provide additional context about data quality issues that may have occurred.
